## Что это за расширение

LRS Twitch Overlay — это расширение для Twitch с внутриигровой справкой для зрителей.

Оно добавляет поверх трансляции отдельную панель, в которой можно:
- открыть список героев;
- посмотреть карточку персонажа;
- открыть словарь мира;
- найти термин через поиск;
- увеличить или уменьшить размер шрифта в открытом термине.

Расширение не требует авторизации, не использует Bits и не зависит от активности чата.
Его можно проверить даже без прямого эфира.

## Что есть в текущей версии

### Главная страница
Теперь расширение начинается с отдельного главного экрана.
На нём есть две крупные кнопки:
- **Герои**
- **Словарь**

### Герои
У зрителей отображается список персонажей и карточки героев.
В карточке есть:
- имя;
- раса;
- класс;
- **архетип**;
- здоровье;
- броня;
- уровень;
- характеристики.

### Словарь
Зрители могут:
- открывать список терминов;
- искать по названию и описанию;
- фильтровать по алфавиту;
- менять размер текста в открытом термине.

### Навигация
На внутренних экранах есть кнопка **«Назад»**, чтобы вернуться на главный экран.

### Темы
В расширении есть **тёмная** и **светлая** тема.
По умолчанию используется тёмная.

## Структура репозитория

### Основные файлы зрительской части
- `index.html` — основная точка входа зрительского overlay.
- `viewer.html` — дополнительная HTML-страница для совместимости со старым путём подключения.
- `viewer.js` — основная логика зрительского интерфейса.
- `scripts.js` — дублирующий JS-файл для обратной совместимости.
- `styles.css` — все стили зрительской части, включая светлую и тёмную тему, адаптив и карточки героев.

### Админка
- `editor.html` — интерфейс административной панели.
- `editor.js` — логика редактирования героев и словаря.
- `admin.css` — стили административной панели.

### Данные
Папка `data/` содержит основные JSON-файлы:
- `heroes.json` — данные по персонажам;
- `glossary.json` — словарь терминов;
- `settings.json` — дополнительные настройки интерфейса.

### Twitch
- `manifest.json` — файл конфигурации расширения для Twitch.

### README-материалы
- `readme_assets/` — изображения для README: логотип, скриншоты интерфейса, скриншоты мобильной версии.

5. Перейти во вкладку **«Словарь»**.
6. Открыть существующий термин.
7. Проверить редактирование текста.
8. Проверить скачивание JSON-файлов.

## Как обновлять данные через админку

Это основной и рекомендуемый способ.

### Ссылка на админку
https://kao820.github.io/twitch-overlay/editor.html

---

## Обновление героев

1. Откройте админку.
2. Перейдите во вкладку **«Герои»**.
3. Нажмите на нужного героя, чтобы открыть форму редактирования.
4. Измените нужные поля:
   - имя;
   - раса;
   - класс;
   - **архетип**;
   - портрет;
   - уровень;
   - здоровье;
   - броня;
   - характеристики;
   - статус персонажа.
5. Нажмите **«Применить»**.
6. После завершения всех изменений нажмите **«Скачать heroes.json»**.
7. Замените старый файл `data/heroes.json` на новый.

---

## Обновление словаря

1. Откройте админку.
2. Перейдите во вкладку **«Словарь»**.
3. Выберите букву или термин, который нужно изменить.
4. Измените:
   - название термина;
   - описание.
5. Нажмите **«Сохранить»**.
6. После завершения всех изменений нажмите **«Скачать glossary.json»**.
7. Замените старый файл `data/glossary.json` на новый.

---

## Что делать после скачивания JSON

После редактирования через админку у вас будут новые версии файлов.

Нужно:
1. Заменить старые JSON-файлы в папке `data/`.
2. Закоммитить изменения в репозиторий.
3. Дождаться обновления GitHub Pages.
4. Проверить изменения по ссылкам:
   - https://kao820.github.io/twitch-overlay/
   - https://kao820.github.io/twitch-overlay/editor.html

## Можно ли редактировать JSON вручную

Да, можно.

Это полезно, если нужно:
- массово менять записи;
- быстро исправить структуру;
- копировать/вставлять блоки данных;
- работать с исходниками напрямую.

### Ручное редактирование героев
Файл:
- `data/heroes.json`

### Ручное редактирование словаря
Файл:
- `data/glossary.json`

### Важно при ручном редактировании
- не ломать JSON-структуру;
- сохранять кавычки и запятые;
- не удалять обязательные поля;
- после изменений проверить файл валидатором JSON, если есть сомнения.
