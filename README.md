# LRS Overlay

Этот репозиторий содержит исходный код Twitch‑расширения «Земли былых легенд: Молчание богов».  Оверлей показывает зрителям справочник героев и словарь терминов, а административная панель позволяет редактировать эти данные без прямого изменения JSON‑файлов.  В новой версии полностью переработан внешний вид, улучшена адаптивность и восстановлена обратная совместимость с ранними версиями манифеста.

## Структура репозитория

В каталоге `docs/` находятся все файлы, которые будет обслуживать GitHub Pages и которые укажет манифест Twitch.  Основные элементы:

- **`index.html`** — главная страница оверлея.  Загружается новым манифестом и подключает `viewer.js` и `styles.css`.
- **`viewer.html`** — fallback‑страница для старых клиентов Twitch.  Загружает тот же HTML, но подключает `scripts.js` вместо `viewer.js`.  Это решение обеспечивает обратную совместимость: старые пользователи продолжат получать актуальный код.
- **`viewer.js`** — основной JavaScript, реализующий логику оверлея: загрузка JSON‑данных, роутинг между экранами, вывод карточек героев и терминов, переключение темы и обработка ошибок изображений.
- **`scripts.js`** — точная копия `viewer.js`.  Оба файла должны оставаться идентичными, чтобы старый манифест и новый загружали одинаковый код.
- **`styles.css`** — полный набор CSS‑правил для оверлея.  Файл содержит переменные для тёмной и светлой тем, описывает макеты главного экрана, списков и карточек, а также оформление модальных окон и шапки.  Все элементы подстраиваются под выбранную тему и корректно отображаются на мобильных устройствах.
- **`editor.html` / `editor.js`** — административная панель.  Позволяет просматривать и редактировать героев и словарь прямо в браузере.  Изменённые данные можно сохранить в виде JSON‑файлов для загрузки в репозиторий.
- **`admin.css`** — стили для админки.  Используются те же переменные, что и в оверлее, поэтому переключатель темы работает одинаково.
- **`data/`** — папка с JSON‑файлами `heroes.json`, `glossary.json` и `settings.json`.  Здесь хранятся данные персонажей, словаря и кастомные настройки цветов.  Изменяйте их вручную или через административную панель.
- **`manifest.json`** — файл манифеста Twitch.  Он указывает на `index.html`, перечисляет JavaScript, CSS и JSON‑файлы и задаёт размеры оверлея.

## Основные возможности

* **Главный экран** — после загрузки пользователи видят приветственный экран с названием проекта и двумя кнопками: «Герои» и «Словарь».
* **Список героев** — персонажи сгруппированы по статусу (живы, мёртвы, неизвестно).  Каждая группа отображается в собственной панели.
* **Карточка героя** — содержит изображение, имя, характеристики (здоровье, броня, уровень), расу, класс и архетип, а также сетку статов.  Карточка адаптируется под тёмную и светлую темы и корректно работает на мобильных устройствах.
* **Словарь** — термины сгруппированы по первой букве.  Есть поиск, фильтр по буквам и кнопки для увеличения и уменьшения шрифта.
* **Переключение темы** — кнопка в шапке (🌙/☀️) переключает тёмную и светлую темы.  Цвета интерфейса задаются через CSS‑переменные, поэтому карточки и панели полностью перекрашиваются, в том числе в админке.
* **Мобильная адаптивность** — верстка основана на flex‑и grid‑модулях.  На маленьких экранах элементы выстраиваются вертикально, а блок с характеристиками героя остаётся в одной строке и позволяет прокручивать содержимое горизонтально.
* **Обратная совместимость** — для старых версий расширения в архиве присутствуют `viewer.html` и `scripts.js`.  Они гарантируют, что даже пользователи, у кого закешированы старые манифесты, автоматически подхватят новую версию.

## Обновление данных

1. **Ручное редактирование.**  Вы можете править файлы `heroes.json`, `glossary.json` и `settings.json` напрямую.  Структура объектов остаётся такой же, как в исходном проекте.
2. **Использование админки.**  Откройте `editor.html` в браузере (на GitHub Pages или локальном сервере).  Выберите вкладку «Герои» или «Словарь», чтобы добавить, изменить или удалить записи.  Кнопки «Сохранить heroes.json» и «Сохранить glossary.json» сохраняют текущие данные в файлы для дальнейшего коммита.
